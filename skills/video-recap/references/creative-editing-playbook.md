# 创作剪辑工作法

本参考只规定 Agent 的创作判断，不增加渲染模块。进入剪辑或写稿前，先完成这里的决策，再写 `clip_plan.json` 或 `narration.json`。

## 决策优先级

硬约束先于审美：

1. **证据真实**：人物、关系、因果和结果必须由画面、对白、可靠背景或用户上下文支撑。
2. **声音可懂**：关键原声不能被截成半句，旁白不能把必须听见的对白完全盖住。

硬约束满足后，发生冲突时按以下顺序取舍：

1. 情绪与人物；
2. 故事推进；
3. 节奏；
4. 视线与画面连续；
5. 技术上的“绝对工整”。

不要为了连续而牺牲人物反应，也不要为了快而删掉真正改变观众感受的瞬间。

## 五次专业判断

### 1. 导演：先决定观众经历什么

先回答，而不是先写开场文案：

- 观众承诺：看完具体会理解、担心或感受到什么？
- POV：这条内容主要跟随谁的经历？谁不是主视角？
- 戏剧问题：观众一直想知道的那个问题是什么？
- 情绪路径：开头的情绪如何变成结尾的情绪？
- 隐瞒与揭示：什么信息暂时不说，何时揭示才最有力量？
- 结尾余味：答案落地后，观众最后应留下什么感受？

至少提出两个可成立的剪辑假设，再选一个。比较的是叙事效果，不是措辞：例如“按主角认知逐步揭示”与“先展示后果再追问原因”。只记录选择与理由，不输出冗长思维过程。

### 2. 故事编辑：把素材组织成变化

beat 不是固定秒数、场景名或一句旁白；beat 是一次可感知的变化。每个保留 beat 至少改变一项：

- 观众知道了什么；
- 人物想要什么；
- 谁占上风；
- 关系如何移动；
- 情绪如何转向；
- 风险或代价如何升级。

如果移除一个 beat 后，因果、人物或情绪都没有损失，它通常不值得占用成片时间。钩子必须提出这条故事真正会兑现的问题，不能制造正文不回答的夸张承诺。

### 3. 画面剪辑：选择时刻，而不只选择事件

对每个 beat 做以下判断：

- **入点**：能否晚一点进入，跳过已经能推断的铺垫？
- **出点**：信息或情绪落地后，能否早一点离开？
- **表演**：说话者与倾听者谁更能承载此刻？动作与反应谁更重要？
- **不可替代瞬间**：停顿、迟疑、失误、眼神、动作声音中，哪一个让人物变得真实？
- **节奏**：这一拍需要冲过去，还是需要多停半秒让含义落地？
- **边界**：优先完整动作、完整台词和自然声音边界；不要机械地把每个片段切成同一长度。

进入 `clip_plan.json` 的不是“发生过的所有重要事件”，而是最能让观众经历这条主线的具体时刻。

### 4. 声音与旁白：先分工，再写字

每个 beat 先指定一个主要 `audio_owner`：

- `original_dialogue`：原对白本身就是事件或表演；
- `action_sound`：动作声音承担冲击、因果或空间感；
- `ambience` / `music`：环境或音乐承担情绪；
- `silence`：沉默、停顿或呼吸是内容；
- `narration`：画面和原声无法独自交代必要的上下文、因果或预期。

只有当旁白有明确工作时才写。`narration_job` 只能从以下任务中选择：

- `context`：补充看不见但必要的背景；
- `causal_link`：连接被压缩掉的因果；
- `foreshadow`：建立稍后会兑现的预期；
- `interpretation`：基于证据指出人物选择、关系或代价；
- `transition`：跨越时间、地点或素材来源；
- `none`：画面、原声或沉默已经足够。

`7:3` 只可作为没有更好判断时的粗略首稿参考，不是质量目标或覆盖配额。强对白可以让一个 beat 完全没有旁白；信息密集的压缩段也可以由旁白主导。

### 5. 观众复核：从第一次观看出发

完成草案后做三次反事实检查：

1. **静音旁白**：只看画面和原声，人物、变化和关键情绪是否仍可感知？
2. **只听声音**：旁白是否形成一条可听懂的主线，而不是字幕腔或画面复述？
3. **第一次观看**：开头承诺了什么、每一拍又打开了什么问题、结尾是否真正回答？

再逐 beat 追问：

- 删除它，什么会坏？什么都不坏就删。
- 换成反应镜头，情绪是否更强？
- 旁白改成原声或沉默，含义是否更好？
- 提前揭示或延后揭示，哪一个更符合 POV？
- 能否晚进、早出，而不损失理解？

只优先处理 1–3 个最高回报修改，避免用大量低价值润色掩盖结构问题。

## 内容质量基线

进入 TTS 前，以下问题都必须能用具体 beat 回答；否则先改计划或脚本：

1. **承诺**：开头提出的真实问题，结尾是否兑现？
2. **主线**：是否有一个明确 POV 和一条主导因果/关系线？
3. **变化**：每个 beat 是否改变知识、权力、目标、关系、情绪或风险？
4. **人物**：是否保留了至少一个不能由剧情概述替代的表演/反应瞬间？
5. **视听分工**：每个 beat 是否明确由画面、原声、沉默或旁白中的谁主导？
6. **必要性**：旁白是否都在增加信息或期待，而不是复述像素？

这套基线与抖音、YouTube 或任何平台数据无关。先让内容本身成立，再讨论标题、封面、时长档位和发布优化。
